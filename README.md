# Daily Survey Telegram Poll Bot

Автоматический опросник для Telegram, работающий полностью на **GitHub Actions**.
Сервер не нужен: скрипт запускается по расписанию, отправляет опрос в указанные чаты и завершает работу.

## 🚀 Быстрая настройка

### 1. Форк репозитория

Нажмите кнопку **Fork** в правом верхнем углу этого репозитория и создайте копию в своём аккаунте.

### 2. Создание бота

1. Откройте Telegram и найдите `@BotFather`
2. Отправьте команду `/newbot`
3. Следуйте инструкциям, задайте имя и юзернейм бота
4. Скопируйте выданный **HTTP API Token** (вида `123456:ABC-DEF...`)
5. Добавьте бота в нужные чаты/группы (должен иметь права на отправку сообщений)

### 3. Настройка Secrets и Variables

Перейдите в ваш форк: `Settings` → `Secrets and variables` → `Actions`

#### 🔒 Вкладка Secrets

| Название    | Значение             | Пример                                 |
| ----------- | -------------------- | -------------------------------------- |
| `BOT_TOKEN` | Токен от BotFather   | `123456789:AAH...`                     |
| `CHAT_IDS`  | JSON-массив ID чатов | `["-1001234567890", "-1009876543210"]` |

> 💡 Как узнать `CHAT_ID`: Добавьте бота в чат, отправьте любое сообщение, затем откройте `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найдите `"chat": {"id": -100...}`.

#### 📦 Вкладка Variables

| Название        | Значение              | Пример                           |
| --------------- | --------------------- | -------------------------------- |
| `POLL_QUESTION` | Текст вопроса         | `Кто сегодня придет на встречу?` |
| `POLL_OPTIONS`  | JSON-массив вариантов | `["да", "нет", "не знаю"]`       |

### 4. Проверка работы

1. Перейдите во вкладку **Actions**
2. Слева выберите `Daily Poll`
3. Справа нажмите **Run workflow** → **Run workflow**
4. Дождитесь зелёной галочки ✅
5. Проверьте указанные Telegram-чаты — опрос должен появиться

## ⏰ Как изменить время отправки

По умолчанию опрос отправляется в `05:00 UTC` (08:00 МСК). Чтобы изменить расписание:

1. Откройте файл `.github/workflows/poll.yml`
2. Найдите строку `cron: "0 5 * * *"`
3. Замените на нужное время в формате **UTC**:  
   `cron: "минута час * * *"`

   | Ваше время (МСК) | Значение cron  |
   | ---------------- | -------------- |
   | 09:00            | `"0 6 * * *"`  |
   | 12:00            | `"0 9 * * *"`  |
   | 18:00            | `"0 15 * * *"` |

4. Закоммитьте изменения. GitHub Actions автоматически подхватит новое расписание.

## 🛠 Локальный запуск (опционально)

Для отладки без деплоя создайте файл `.env` в корне репозитория:

```env
BOT_TOKEN=ваш_токен
CHAT_IDS=["-100123456789"]
POLL_QUESTION="Кто сегодня придет?"
POLL_OPTIONS=["да","нет","хз"]
```

Установите зависимости и запустите:

```bash
pip install -r requirements.txt
python main.py
```

## 📝 Примечания

- Опрос **не анонимный** (`is_anonymous=False`), чтобы видели, кто именно голосовал.
- Скрипт отправляет опросы последовательно. При ошибке в одном чате процесс не прерывается.
- GitHub Actions использует часовой пояс **UTC**. Не забудьте конвертировать своё время.
- Репозиторий не требует серверов, хостинга или Docker. Всё работает на инфраструктуре GitHub.
